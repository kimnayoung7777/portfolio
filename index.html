2. 핵심 설계 포인트

  • 권한 구조 (Authorization): 모임 주최자, 댓글 작성자, 관리자 권한 분리. 기능 단위가 아니라 역할 기반 접근 제어(RBAC)로 설계
  • 데이터 라이프사이클: Soft Delete 기반 데이터 관리. 삭제 시 데이터 물리 제거가 아닌 상태값 변경으로 데이터 추적 가능성 유지
  • 계층형 댓글 구조: 부모 댓글 및 대댓글 구조 설계. 삭제 시 구조 유지 로직 구현

3. 트러블슈팅 및 해결 방식

🔥 문제 상황

운영자 관리 부재: 초기에는 작성자만 댓글 삭제가 가능해 부적절한 댓글 관리가 불가능

계층 구조 붕괴: 부모 댓글 삭제 시, 대댓글이 독립적으로 남아 구조가 깨질 위험성 존재

UX 혼란: 삭제된 댓글을 완전히 제거할 경우, 사용자가 이전 맥락을 인지할 수 없음

💡 해결 방식

권한 기반 삭제 정책 분리: 작성자 삭제 시 (본인+하위 댓글 전체 삭제), 모임장 삭제 시 (데이터 유지+안내 문구 처리)

계층형 데이터 처리: WHERE commentId = ? OR parentCommentId = ? 쿼리로 부모 삭제 시 자식 댓글까지 일괄 처리

Soft Delete 적용: 물리 삭제 대신 상태값 기반 논리적 삭제 적용 및 조회 조건 분기 처리

4. 결과 및 핵심 키워드

운영자 댓글 관리 구조를 설계하고, 계층형 구조 유지 및 데이터 무결성을 확보하여 사용자 UX를 개선했습니다.

3. 트러블슈팅 및 해결 방식

🔥 문제 상황

매출 데이터 정합성: 주문 데이터 기준 통계를 단순 합산할 경우 취소/미완료 주문이 포함되는 문제 발생

시각화 비율 오차: 차트 비율 계산 시 소수점 반올림으로 인해 전체 합이 100%가 되지 않는 부동소수점 오차 발생 (예: 99.9%)

💡 해결 방식

데이터 정합성 개선: order_status = 5 (배달 완료) 데이터만 집계하여 불필요한 데이터를 원천 차단하고 결제 기준 반영

비율 오차 보정: 정수 기반 비율 계산 후 오차 값을 별도 산출하고, 편차가 큰 항목부터 재분배하여 합계 100%를 강제 보정

4. 실제 구현 결과

결제 확정 데이터 기반의 매출 집계 로직이 적용된 실제 관리자 대시보드 화면입니다. 매장의 월별 매출 추이뿐만 아니라 배달 경로 별 비중을 직관적으로 시각화했습니다. 특히 반올림 오차 보정 알고리즘을 통해 모든 수치의 총합이 정확히 100%로 일치하도록 구현하여 데이터 신뢰도를 높였습니다.

5. 핵심 키워드

실제 결제 기준의 통계를 제공하여 데이터 정합성을 확보하고 시각화 정확도를 개선하여 관리자 대시보드의 신뢰성을 향상시켰습니다.
